# Achievement Acceleration Engine

This document outlines a sustainable, realistic, and non-spammy cadence for scaling GitHub achievements.

## Core Milestones Targets
*   **Pull Shark:** Merged Pull Requests (Targets: 16, 128, 1024). Requires sustained velocity in active repos.
*   **Open Sourcerer:** Approved PRs in repositories you don't own.
*   **Pair Extraordinaire:** Co-authored commits.
*   **Galaxy Brain:** Accepted answers in GitHub Discussions.
*   **Starstruck:** Stars on owned repositories (Targets: 16, 128, 512).

---

## The Velocity Plan

### Daily Actions (15-30 Minutes)
*   **Discussions (Galaxy Brain):** Spend 10 minutes answering questions in active repositories (e.g., Next.js, Flask, Tailwind). Focus on providing code-snippet solutions.
*   **Triage:** Spend 5 minutes checking notifications to respond quickly to maintainer feedback on your open PRs.
*   **Micro-Commit:** Push at least one meaningful commit to a personal project (e.g., `dev-utility-lab` or a micro-repo) to maintain the streak.

### Weekly Quotas (2-4 Hours)
*   **Open Source PRs (Pull Shark/Open Sourcerer):** Submit 2-3 high-quality PRs to targeted repositories (see `OSS_TARGETS.md`). Focus on tests, docs, or bug fixes.
*   **Pair Programming (Pair Extraordinaire):** Reach out to a peer or an open-source collaborator contextually. Agree to co-author a complex feature or documentation overhaul. Use the exactly formatted `Co-authored-by: name <email>` trailer in commit messages.
*   **Ecosystem Maintenance:** Create and merge 1-2 PRs within your own micro-repo ecosystem (merging from feature branches to main).

### Monthly Review
*   **Portfolio Audit:** Archive dead projects. Update READMEs on active projects.
*   **Achievement Check:** Readjust PR targets based on merge rates. If PRs are stagnating, pivot to faster-moving repositories.

---

## Strategy Details

### Scaling Pair Extraordinaire Intelligently
1.  **The "Docs and Code" Split:** If contributing to a mid-sized OSS repo, pair with another contributor. One writes the tests, the other writes the function. Co-author the commit.
2.  **Internal Ecosystem:** When building the micro-repos, operate using secondary organizational accounts or via authorized collaborators to simulate a team environment transparently.

### Scaling Galaxy Brain Intelligently
1.  **Find Niche Frameworks:** Mega-frameworks answer questions too quickly. Find libraries with 2k-5k stars that have active discussion boards but fewer "power users."
2.  **Provide Reproducible Examples:** Never just answer with text. Provide a small code block or a link to a minimal reproduction repo. This guarantees accepted answers.

### Maintaining Realism
*   **Cadence Variations:** Don't perfectly merge 3 PRs every week. Humans take breaks. Allow natural dips in activity.
*   **Meaningful Commit Messages:** Never use "update" or "fix". Use conventional commits (`fix: correct off-by-one error in pagination`).
